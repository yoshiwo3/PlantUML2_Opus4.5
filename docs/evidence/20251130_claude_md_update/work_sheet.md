## 学んだ教訓

1. **auto-compact後は必ず確認**
   - 前セッションの内容を鵜呑みにしない
   - SERENAメモリやファイルで実際の状況を確認

2. **「完了」の定義を明確に**
   - 下書き（design/）の存在 ≠ 完了
   - 正式版（proposals/）の存在 = 完了

3. **報告前に検証**
   - ファイルパスを主張する前に実際に読む
   - grep検索で行番号を確認

4. **目的を理解してから作業**
   - なぜその作業をするのか
   - わからなければ質問する
