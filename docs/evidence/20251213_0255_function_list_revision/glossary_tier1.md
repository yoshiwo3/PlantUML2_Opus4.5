## 🔰 基本用語（15語）

### 1. ログイン / ログアウト

| 区分 | 内容 |
|------|------|
| **一言で** | アプリに「自分です」と名乗ること（ログイン）、「おしまい」と伝えること（ログアウト） |
| **例えると** | 会社の入館証をかざして入ること（ログイン）、退館時にかざすこと（ログアウト） |
| **なぜ必要？** | 誰がどのデータを見れるかを区別するため。ログインしないと、他の人の図表が見えたり、自分の図表が他人に見られたりしてしまう |
| **本資料での登場** | F-AUTH-01, F-AUTH-02 |

---

### 2. セッション

| 区分 | 内容 |
|------|------|
| **一言で** | ログインしてからログアウトするまでの「ひとつながりの作業時間」 |
| **例えると** | カフェで席に座ってから帰るまでの時間。席を確保している間だけ注文できる |
| **なぜ必要？** | 毎回パスワードを入力しなくても「この人はさっきログインした人だ」と覚えておくため |
| **本資料での登場** | F-AUTH-01, F-AUTH-02, 共通エラー（SESSION_EXPIRED等） |

**補足**: セッションには有効期限があります（PlantUML Studioでは7日間）。期限が切れると再ログインが必要です。

---

### 3. プロジェクト

| 区分 | 内容 |
|------|------|
| **一言で** | 複数の図表をまとめて管理するための「フォルダ」のようなもの |
| **例えると** | 仕事の案件ごとに分けたフォルダ。「A社システム開発」「社内研修資料」など |
| **なぜ必要？** | 図表が増えると探しにくくなる。目的別にグループ化することで整理整頓 |
| **本資料での登場** | F-PRJ-01〜04 |

**例**:
```
私のプロジェクト/
├── ECサイト設計/
│   ├── ユースケース図.puml
│   ├── クラス図.puml
│   └── 画面フロー.excalidraw
└── 社内システム/
    └── 業務フロー図.puml
```

---

### 4. 図表（ダイアグラム）

| 区分 | 内容 |
|------|------|
| **一言で** | PlantUMLやExcalidrawで作成する1つの図（ファイル） |
| **例えると** | 1枚の設計図。PowerPointの1スライドに相当 |
| **なぜ必要？** | システムの構造や流れを視覚的に表現することで、言葉だけでは伝わりにくい内容を明確にする |
| **本資料での登場** | F-DGM-01〜11 |

**種類**:
- **PlantUML図表**（`.puml`）: テキストで書いて図に変換するタイプ
- **Excalidraw図表**（`.excalidraw.json`）: マウスで描くタイプ

---

### 5. テンプレート

| 区分 | 内容 |
|------|------|
| **一言で** | 図表を作るときの「ひな形」。ゼロから作らずに済む |
| **例えると** | 履歴書のフォーマット。枠組みが決まっていて、中身を埋めるだけ |
| **なぜ必要？** | 毎回ゼロから作ると時間がかかる。定型的な図は用意されたテンプレートを使うと効率的 |
| **本資料での登場** | F-DGM-02 |

**例**: シーケンス図テンプレート、クラス図テンプレート、ワイヤーフレームテンプレート等

---

### 6. 保存

| 区分 | 内容 |
|------|------|
| **一言で** | 作業中の図表をクラウド上に記録すること |
| **例えると** | Wordの「保存」と同じ。保存しないと編集内容が消える |
| **なぜ必要？** | ブラウザを閉じたり、PCが固まったりしても、保存していれば作業内容が残る |
| **本資料での登場** | F-DGM-05 |

**注意点**: PlantUML Studioは**手動保存**です（Ctrl+S または保存ボタン）。自動保存ではないので、こまめに保存しましょう。

---

### 7. エクスポート

| 区分 | 内容 |
|------|------|
| **一言で** | 図表を別の形式（PNG画像、SVGファイル等）でダウンロードすること |
| **例えると** | Excelファイルを「PDF形式で保存」するようなもの |
| **なぜ必要？** | 作った図表を資料に貼り付けたり、他のツールで使ったりするため |
| **本資料での登場** | F-DGM-06 |

**対応形式**:
| 形式 | 用途 |
|------|------|
| PNG | 画像として貼り付け（PowerPoint、Word等） |
| SVG | 拡大しても綺麗（Web、印刷用） |
| PDF | ドキュメントとして保存 |

---

### 8. プレビュー

| 区分 | 内容 |
|------|------|
| **一言で** | 書いたコードがどんな図になるか、保存前に確認すること |
| **例えると** | 印刷前の「印刷プレビュー」 |
| **なぜ必要？** | PlantUMLはテキストで書くので、実際の見た目を確認しながら調整するため |
| **本資料での登場** | F-DGM-04 |

**補足**: 入力を止めて0.5秒後に自動的にプレビューが更新されます。

---

### 9. バリデーション（検証）

| 区分 | 内容 |
|------|------|
| **一言で** | 入力内容が正しいかどうかチェックすること |
| **例えると** | 郵便番号欄に「あいうえお」と入力すると「数字を入力してください」と言われるアレ |
| **なぜ必要？** | 間違った入力を早めに検出して、後で困らないようにするため |
| **本資料での登場** | 各機能のエラーケース |

**例**: プロジェクト名に使えない文字（`/` `\` 等）を入力すると、保存前にエラーを表示

---

### 10. エラー

| 区分 | 内容 |
|------|------|
| **一言で** | 操作がうまくいかなかった時にシステムが出すメッセージ |
| **例えると** | ATMで残高不足の時に「残高が不足しています」と表示されるもの |
| **なぜ必要？** | 何が問題なのか、どうすればいいのかをユーザーに伝えるため |
| **本資料での登場** | 全機能のエラーケース、共通エラーコード |

**エラーの読み方**:
1. メッセージを読む（何が起きたか）
2. 対処法を確認する（どうすればいいか）
3. それでも解決しなければサポートに連絡

---

### 11. OAuth（オーオース）

| 区分 | 内容 |
|------|------|
| **一言で** | 他のサービス（GitHub/Google）のアカウントを借りてログインする仕組み |
| **例えると** | ホテルで「会社の社員証をお持ちですか？」と聞かれ、見せると部屋に入れるようなもの |
| **なぜ必要？** | サービスごとにパスワードを作ると覚えきれないし、漏洩リスクも増える |
| **本資料での登場** | F-AUTH-01 |

**仕組み（簡略版）**:
```
あなた → 「このアプリを使いたい」
   ↓
アプリ → 「GitHubさん、この人は本物ですか？」
   ↓
GitHub → あなたに「このアプリにアクセスを許可しますか？」
   ↓
あなた → 「許可します」
   ↓
GitHub → アプリに「この人は本物です」
   ↓
アプリ → 「ようこそ！」
```

---

### 12. ダッシュボード

| 区分 | 内容 |
|------|------|
| **一言で** | ログイン後に最初に表示される「ホーム画面」 |
| **例えると** | スマホのホーム画面。よく使うアプリが並んでいる |
| **なぜ必要？** | プロジェクト一覧、最近の図表、クイックアクセスなど、作業の起点となる場所 |
| **本資料での登場** | F-AUTH-01（ログイン後の遷移先） |

---

### 13. ユーザー

| 区分 | 内容 |
|------|------|
| **一言で** | このアプリを使う人。あなたのこと |
| **例えると** | サービスの会員 |
| **なぜ必要？** | 誰のデータかを区別するため |
| **本資料での登場** | 全機能の「主アクター」 |

**種類**:
| 種類 | 説明 |
|------|------|
| エンドユーザー | 一般的な利用者（図表を作る人） |
| 開発者（管理者） | システムの設定を管理する人 |

---

### 14. 一覧 / 一覧表示

| 区分 | 内容 |
|------|------|
| **一言で** | 複数の項目をリスト形式で表示すること |
| **例えると** | メールの受信トレイ。件名が縦に並んでいる |
| **なぜ必要？** | たくさんあるプロジェクトや図表から目的のものを探すため |
| **本資料での登場** | F-PRJ-02, F-DGM-01 |

---

### 15. 削除

| 区分 | 内容 |
|------|------|
| **一言で** | プロジェクトや図表を完全に消すこと |
| **例えると** | ゴミ箱を空にする。元には戻せない |
| **なぜ必要？** | 不要になったデータを整理するため |
| **本資料での登場** | F-PRJ-04, F-DGM-09 |

**注意**: PlantUML Studioの削除は**取り消し不可**です。削除前に確認ダイアログが表示されるので、よく読んでから「削除」を押してください。
