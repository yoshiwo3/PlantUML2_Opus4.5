# UI設計 パターンチェックリスト

**作成日**: 2025-12-22
**バージョン**: 1.0
**目的**: UI設計図表作成時に設計品質を確保するためのチェックリスト

---

## 概要

本チェックリストは、TD-015準拠（低精度、手書き風、グレースケール）とは別に、**UIパターンの観点**からワイヤーフレームを検証するためのものです。

**背景**: UI設計図表開発憲法 v3.2 §8で定義された19項目のUIパターンチェックを、より詳細に展開したものです。

---

| タイミング | 使用するチェックリスト |

## 1. 画面特性分析（作成前に実施）

対象画面の特性を分析し、必要なUIパターンを特定する。

| # | 質問 | 回答 | 該当時の必須パターン |
|:-:|------|:----:|---------------------|
| 1 | フォーム入力があるか？ | Yes/No | フォームパターン（§4） |
| 2 | 一覧表示があるか？ | Yes/No | ナビゲーションパターン（§3） |
| 3 | モーダル/ダイアログか？ | Yes/No | モーダルパターン（§5.1） |
| 4 | 管理画面か？ | Yes/No | テーブルパターン（§5.2） |
| 5 | エラー表示が必要か？ | Yes/No | フィードバックパターン（§5.3） |
| 6 | 主要アクションがあるか？ | Yes/No | CTA配置パターン（§3.3） |

## 2. アクセシビリティパターン（必須確認）

WCAG 2.1 Level AA準拠の基本要件を確保する。

### チェックリスト

- [ ] **AP-001**: フォーカス順序が論理的か？
  - 左上→右下、Tab順序が自然
- [ ] **AP-002**: インタラクティブ要素の最小サイズは44x44pxか？
  - ボタン、リンク、入力フィールド
- [ ] **AP-003**: 色だけに依存した情報伝達をしていないか？
  - エラー: 色 + テキスト + アイコン
- [ ] **AP-004**: テキストコントラストは十分か？
  - 通常テキスト: 4.5:1以上
  - 大きいテキスト: 3:1以上
- [ ] **AP-005**: フォームにラベルがあるか？
  - 全入力フィールドにラベル必須

### AP-001: フォーカス順序パターン

```
正しい順序:
  1. ヘッダーナビゲーション
  2. メインコンテンツ（上→下、左→右）
  3. サイドバー
  4. フッター

禁止:
  - ビジュアル順序とTab順序の不一致
  - 隠れた要素へのフォーカス
```

### AP-003: 色に依存しない情報伝達

```
❌ 悪い例:
  - 赤い枠 = エラー（色のみ）

✅ 良い例:
  - 赤い枠 + "!"アイコン + エラーメッセージテキスト
```

## 3. ナビゲーションパターン（必須確認）

ユーザーがサイト内を迷わず移動できることを確保する。

### チェックリスト

- [ ] **NP-001**: 現在位置が明確か？
  - パンくずリスト、アクティブメニュー表示
- [ ] **NP-002**: 主要ナビゲーションが一貫しているか？
  - 全画面で同じ位置
- [ ] **NP-003**: 戻る/キャンセル操作が明確か？
  - 「←戻る」ボタン、ESCキー対応（モーダル）
- [ ] **NP-004**: 階層構造が3レベル以内か？
  - 深すぎる階層は避ける

### NP-001: 現在位置の明示

```
パンくずリスト例:
  ホーム > プロジェクト一覧 > プロジェクトA > 図表エディタ

アクティブメニュー例:
  ダッシュボード | [プロジェクト] | 設定 | ヘルプ
                    ↑ 太字/下線/背景色変更
```

### NP-003: CTA（Call To Action）配置パターン

```
主要アクション配置:
  - 右上/右下に主要ボタン
  - 左にキャンセル/戻るボタン

ボタン優先度:
  1. Primary: 最も重要（濃いグレー背景）
  2. Secondary: 補助的（枠線のみ）
  3. Tertiary: 最小限（テキストリンク）
```

## 4. フォームパターン（必須確認）

ユーザーが正しく入力できることを確保する。

### チェックリスト

- [ ] **FP-001**: 入力フィールドにラベルがあるか？
  - プレースホルダーのみは不可
- [ ] **FP-002**: 必須項目が明示されているか？
  - 「*」マークまたは「(必須)」表示
- [ ] **FP-003**: エラー表示がフィールド近くにあるか？
  - フィールド直下にエラーメッセージ
- [ ] **FP-004**: 入力ヒント/例が必要か？
  - 複雑な入力形式には例を表示
- [ ] **FP-005**: 送信ボタンの文言が具体的か？
  - 「送信」より「アカウントを作成」

### FP-001: ラベル配置パターン

```
推奨: 上ラベル
  ┌────────────────┐
  │ メールアドレス  │
  ├────────────────┤
  │                │
  └────────────────┘

許容: 左ラベル（横幅に余裕がある場合）
  メールアドレス  ┌────────────────┐
                 │                │
                 └────────────────┘
```

### FP-003: エラー表示パターン

```
ワイヤーフレームでのエラー表示:
  ┌─────────────────────────┐
  │ メールアドレス *        │
  ├─────────────────────────┤
  │ [入力フィールド]        │
  ├─────────────────────────┤
  │ ! 有効なメールアドレスを │
  │   入力してください       │
  └─────────────────────────┘
```

## 5. 特殊UIパターン（状況に応じて確認）

### 5.1 モーダル/ダイアログパターン

- [ ] **MP-001**: モーダルタイトルが明確か？
- [ ] **MP-002**: 閉じるボタン（×）があるか？
- [ ] **MP-003**: ESCキーで閉じることを想定しているか？
- [ ] **MP-004**: フォーカストラップを考慮しているか？
- [ ] **MP-005**: 背景オーバーレイがあるか？

```
モーダル基本構造:
  ┌─────────────────────────────┐
  │ [タイトル]            [×]  │
  ├─────────────────────────────┤
  │                             │
  │ [コンテンツ]               │
  │                             │
  ├─────────────────────────────┤
  │     [キャンセル] [確定]    │
  └─────────────────────────────┘
```

### 5.2 テーブル/一覧パターン（管理画面）

- [ ] **TP-001**: カラムヘッダーが明確か？
- [ ] **TP-002**: ソート機能を想定しているか？
- [ ] **TP-003**: ページネーションがあるか？
- [ ] **TP-004**: 行アクション（編集/削除）が明確か？
- [ ] **TP-005**: 空状態（データなし）を考慮しているか？

```
テーブル基本構造:
  ┌──────┬──────────┬────────┬─────────┐
  │ ID   │ 名前     │ 作成日 │ 操作    │
  ├──────┼──────────┼────────┼─────────┤
  │ 001  │ プロジェ │ 12/20  │ [編集]  │
  │ 002  │ クトA    │ 12/21  │ [削除]  │
  └──────┴──────────┴────────┴─────────┘
  [<] 1 2 3 ... 10 [>]  ← ページネーション
```

### 5.3 フィードバックパターン

- [ ] **FB-001**: 成功メッセージの表示位置を想定しているか？
- [ ] **FB-002**: エラーメッセージの表示位置を想定しているか？
- [ ] **FB-003**: ローディング状態を想定しているか？
- [ ] **FB-004**: 確認ダイアログが必要な操作を特定しているか？

```
フィードバック位置:
  - グローバル: 画面上部（トースト/バナー）
  - インライン: 該当フィールド直下
  - モーダル: 重要な確認時
```

## 6. レスポンシブ考慮（推奨確認）

ワイヤーフレームは1サイズで良いが、レスポンシブ対応の考慮点を記録する。

### チェックリスト

- [ ] **RP-001**: モバイルでのナビゲーション方式を想定しているか？
  - ハンバーガーメニュー、ボトムナビ
- [ ] **RP-002**: テーブルのモバイル表示を想定しているか？
  - カード形式、横スクロール
- [ ] **RP-003**: タッチ操作を想定しているか？
  - タップターゲット48x48px以上
