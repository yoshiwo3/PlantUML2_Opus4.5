## 概要

シーケンス図全14本完成後、Phase 4（画面遷移図・ワイヤーフレーム）の作業準備として、UI設計図表開発憲法を作成。

---

## 2成果物方式

| 成果物 | ツール | 目的 | 作成順序 |
|--------|--------|------|:--------:|
| **ワイヤーフレーム** | Excalidraw | 各画面のレイアウト定義 | 1st |
| **画面遷移図** | PlantUML state | 画面間の遷移ロジック定義 | 2nd |

**理由**: ワイヤーフレームを先に作成することで、画面遷移図のノードが確定する

---

## TD-015原則（ワイヤーフレーム）

| 原則 | 説明 |
|:----:|------|
| **低精度** | Low-Fidelity（低忠実度）が正解。ピクセルパーフェクトは不要 |
| **手書き風** | 要件変更の柔軟性を示す。シャープなデザインは設計フェーズ以降 |
| **グレースケール** | 色の議論を避ける。色は設計フェーズで決定 |

> 参照: `.serena/memories/wireframe_research_td015_decision_2025-11-26_29.md`

## PlantUML state diagram

Context7確認結果: **重大な構文制限なし**（シーケンス図より単純）

| 機能 | 構文 |
|------|------|
| 開始点 | `[*] --> State` |
| 終了点 | `State --> [*]` |
| 遷移 | `State1 --> State2 : ラベル` |
| 分岐 | `<<choice>>` |
| 手書き風 | `!option handwritten true` |
