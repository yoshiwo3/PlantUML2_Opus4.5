# 機能一覧表の導入決定

## 決定日: 2025-12-06

## 背景

プロジェクト作業中に「機能一覧を作成した方が良いのでは？」という提案があり、検討の結果、機能一覧表を導入することを決定。

## 選択肢と決定

### 検討した選択肢

| 選択肢 | 内容 |
|-------|------|
| A | 機能一覧表と業務フロー・DFD対比表を両方作成 |
| B | 機能一覧表のみ作成 |
| **C（採用）** | 機能一覧表に業務フロー・DFD対比を統合 |

### 決定: 選択肢C

**機能一覧表（業務フロー・DFD対比含む）** として1つのドキュメントに統合。

## 機能一覧表の構成

| 項目 | 内容 |
|------|------|
| 機能ID | F-001〜 |
| 機能名 | 機能の名称 |
| 概要 | 機能の説明 |
| 対応UC | UC番号との紐付け |
| 優先度 | MVP / Phase 2 / Phase 3 |
| 技術要件 | 使用技術・API |
| 依存機能 | 前提となる機能 |
| 画面 | 関連画面名 |
| API | 関連APIエンドポイント |
| 業務フロー | 対応する業務フロー番号 |
| DFD | 対応するDFD番号 |
| 整合性 | ✅ / ❌ |
| 受け入れ条件 | 完了の定義 |

## 作成タイミングと役割

### 作成順序
```
業務フロー図（残り3.7, 3.8）
    ↓
データフロー図（DFD）
    ↓
機能一覧表  ← 業務フロー・DFD対比を含む
    ↓
【レビュー・修正】← 不備発見、フロー修正
    ↓
クラス図
```

### 機能一覧表の役割
- 機能の網羅性確認
- 業務フローとDFDの整合性チェック
- 漏れ・重複の発見
- フロー修正のトリガー
